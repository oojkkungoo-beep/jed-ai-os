# Checklist การเพิ่ม Agent ใหม่เข้าทีม Laura

**บังคับใช้ทุกครั้ง** ที่จะเพิ่ม agent คนใหม่ — Vera เป็นเจ้าของ checklist นี้ ตรวจให้ครบก่อนประกาศว่า agent ใหม่ "พร้อมใช้งาน" (เพิ่ม 2026-06-21 หลังเจอว่า Cinder สร้างรอบแรกยังไม่ครบ)

## ขั้น 0 — ต้องมี "ช่องว่างจริง" ก่อน
ห้ามเพิ่ม agent เพราะ "น่าจะดี" — ต้องมาจาก Vera audit เจอช่องว่างจริง หรือ Jed ชี้ปัญหาที่มีหลักฐาน (ดู [[project_devil_gemini]] เป็นตัวอย่าง)

## ขั้น 1 — ตั้งชื่อ
- [ ] ชื่อเดี่ยว ออกเสียงง่าย ไม่ใช่รูปแบบ "[ชื่อเดิม]-[คำต่อท้าย]" (Jed ปฏิเสธ "Forge-Ops" ไว้ชัดแล้ว — ดู [[project_org_structure_cinder]])
- [ ] ชื่อสื่อบทบาทแบบเป็นนัย ไม่ตรงตัวจนไม่มีตัวตนให้สร้าง character ได้
- [ ] เพศ + คำลงท้าย (ครับ/นะครับ หรือ ค่ะ/นะคะ) ชัดเจน ไม่ขัดกับชื่อ

## ขั้น 2 — ไฟล์ `team/[ชื่อ].md` ต้องมีครบทุกหัวข้อ
- [ ] Gender + คำลงท้าย
- [ ] Role (1 บรรทัด)
- [ ] Model แนะนำ + เหตุผล (sync เข้า `team/model_assignment.md` ตารางหลักด้วย)
- [ ] 🌍 World-Class Standard — เทียบกับตำแหน่งมืออาชีพระดับโลกจริงที่มีอยู่ (ไม่ใช่คำกว้างๆ)
- [ ] แผนก + หัวหน้าแผนก (sync เข้า `team/org_structure.md`)
- [ ] Trigger keyword + เส้นแบ่งกับ agent ที่งานใกล้เคียงกัน (ถ้ามีจุดที่สับสนได้)
- [ ] Output Format (โครงสร้างคำตอบตายตัว)
- [ ] กฎการทดสอบ/ตรวจสอบงานของตัวเอง
- [ ] **Self-Reflection Loop** — คำถามเช็คตัวเองก่อน submit (เคยพลาดในรอบ Cinder แรก — ห้ามลืมอีก)
- [ ] Security Rules (ถ้างานแตะโค้ด/ข้อมูล)
- [ ] **Domain knowledge ที่มีอยู่แล้วในทีม** — เช็คว่ามีความรู้ปฏิบัติจริง (เช่น deploy rules, GAS rules) ที่ "ควรเป็นของ agent ใหม่นี้" อยู่ในไฟล์ agent อื่นไหม ถ้ามีให้ย้าย/sync เจ้าของให้ตรงบทบาทจริง ไม่ปล่อยซ้ำซ้อนหรือขาดหาย
- [ ] บันทึก Output path (`output/[ประเภท]/YYYY-MM-DD-...`)

## ขั้น 3 — ไฟล์ `characters/[ชื่อ].md`
- [ ] เผ่าพันธุ์/อายุ/เพศ + ลักษณะภายนอกที่สื่อบทบาทผ่านสัญลักษณ์
- [ ] นิสัย + น้ำเสียง ตรงกับ team file
- [ ] Job Description code block (สรุปย่อจาก team file)
- [ ] Image Prompt code block — card layout แบบเดียวกับทุกคน (vertical card, ชื่อมุมล่างซ้าย, badge pill มุมบนซ้าย, cream background)

## ขั้น 4 — เชื่อมเข้าระบบ
- [ ] `CLAUDE.md` routing table เพิ่มบรรทัดใหม่
- [ ] `team/model_assignment.md` ตารางหลักเพิ่มแถว
- [ ] `team/org_structure.md` เพิ่มในแผนกที่ถูกต้อง
- [ ] ถ้า agent ใหม่ "รับช่วงงาน" จาก agent เดิม (เช่น Cinder รับ deploy จาก Forge) → ไปแก้ไฟล์ agent เดิมให้ระบุชัดว่าโอนความรับผิดชอบไปแล้ว ห้ามปล่อยให้ทั้งสองไฟล์อ้างว่าตัวเองเป็นเจ้าของเรื่องเดียวกัน

## ขั้น 5 — Vera ตรวจซ้ำก่อนปิดงาน
หลังสร้างครบ Vera ต้องถามตัวเอง: **"ถ้า agent นี้รับงานจริงพรุ่งนี้ มีอะไรที่จะทำให้ตอบไม่ได้/ทำพลาดเพราะข้อมูลไม่ครบไหม"** — ถ้าตอบไม่ได้แน่ใจ ให้เติมก่อนประกาศว่า "พร้อมใช้งาน" อย่าปล่อยให้ Jed เป็นคนเจอช่องโหว่เอง (เหมือนที่เกิดกับ Cinder รอบแรก)
