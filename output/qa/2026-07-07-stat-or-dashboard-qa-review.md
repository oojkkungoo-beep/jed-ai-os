# QA Review — STAT_OR Dashboard (ระบบวิเคราะห์สถิติห้องผ่าตัด)

Dashboard สถิติเคสผ่าตัด ~15,959 เคส (ม.ค. 2568 - ก.ค. 2569) สร้างโดยทีม Angie
รันด้วย Vite dev server พอร์ต 4999 ดึงข้อมูลสดจาก Google Sheets + สำรอง data.csv ในเครื่อง

## 🔴 CRITICAL — ยังไม่แก้ รอ Jed ตัดสินใจ

### Google Sheet ทะเบียนผ่าตัดเปิดสาธารณะ (ยืนยันแล้วด้วยการทดสอบจริง)
- ทดสอบดาวน์โหลด URL export โดยไม่ล็อกอิน → ได้ **HTTP 200 ข้อมูลผู้ป่วยครบทุกเคส**
  (ชื่อ-สกุล, HN, AN, การวินิจฉัย, สิทธิ์การรักษา, ทีมแพทย์)
- Sheet ID ฝังอยู่ใน index.js → ใครได้ไฟล์โปรเจกต์/เปิด DevTools ก็เห็น URL และดึงข้อมูลทั้งฐานได้
- นี่คือข้อมูลสุขภาพ = ข้อมูลอ่อนไหวตาม PDPA มาตรา 26 — เสี่ยงสูงมากถ้าหลุด
- สถาปัตยกรรม "live sync" ของ Angie **บังคับ** ให้ sheet ต้องเปิดสาธารณะ จึงเป็น trade-off เชิงนโยบาย

**ทางเลือก (เรียงตามที่แนะนำ):**
1. **ปิด sheet เป็น private + ใช้ data.csv ในเครื่องอย่างเดียว** — ปลอดภัยสุด แลกกับต้องอัปเดต csv เอง (ทำสคริปต์ sync ให้ได้)
2. **สร้าง sheet สำเนาแบบตัดตัวระบุตัวตน** (ตัดชื่อ, HN, AN ออก) เปิดสาธารณะเฉพาะสำเนา — ยังได้ live sync แต่ค้นหารายคนไม่ได้
3. **GAS Web App เป็นตัวกลาง + token** — ได้ทั้ง live + ปลอดภัยระดับหนึ่ง (ใช้ pattern จาก knowledge_gas_sheets_webapp)

## 🟡 ประเด็นที่ "ไม่แก้เอง" — ต้องการคำยืนยันจาก Jed

1. **เคสที่ไม่มีเวลา Time Room In ถูกนับเป็น "นอกเวลาราชการ" ทั้งหมด** — ข้อมูลจริงมีหลายแถวที่ช่องนี้ว่าง สถิติ OT จึงสูงเกินจริง ควรตัดสินว่า: นับแยกเป็น "ไม่ระบุ" หรือใช้เวลาอื่น (เช่น Time Op.) แทน
2. **KPI "อัตราเคสฉุกเฉิน" นับเฉพาะคำว่า Emergency (NEST 1-2)** ไม่รวม Urgent NEST 3-4 — ตรงเจตนาหรือไม่
3. **URL ดึงข้อมูลใช้พารามิเตอร์ `sheet=ชื่อแผ่น` ซึ่ง Google ไม่รองรับจริง** — ตอนนี้ได้แผ่นแรกโดยบังเอิญ ถ้ามีคนสลับลำดับแผ่นใน spreadsheet ข้อมูลจะเพี้ยนเงียบๆ ควรเปลี่ยนเป็น `gid=` (ต้องรู้ gid ของแผ่นทะเบียนผ่าตัด)
4. **เคส 4 แถวหายจากหน้าจอ** (15,959 → 15,955) เพราะวันที่ parse ไม่ได้/ไม่ใช่ปี 2025-2026 — ควรตรวจข้อมูลต้นทาง
5. **รูปภาพในหน้ารายละเอียดเคสไม่มีทางแสดง** — ข้อมูลชี้โฟลเดอร์ `ทะเบียนผ่าตัด_Images/` ซึ่งไม่มีในโปรเจกต์ ต้อง copy โฟลเดอร์มาไว้ที่ public/ ถ้าอยากใช้ฟีเจอร์นี้
6. **เอกสาร Angie ระบุ "ไม่ต้องเปิดคอมทิ้งไว้" — คลาดเคลื่อน** จริงๆ ระบบรันบน localhost เครื่องใครเครื่องมัน ทุกเครื่องที่ใช้ต้องติดตั้ง Node + npm install เอง
7. **โฟลเดอร์นี้มีไฟล์ PHI เต็มรูปแบบ** (data.csv 14MB + ทะเบียนผู้ป่วยผ่าตัด.xlsx 5MB) — ยังไม่มี git (ดี) **ห้าม** init git แล้ว push ขึ้น remote ใดๆ เด็ดขาด และควรระวังการแชร์โฟลเดอร์
