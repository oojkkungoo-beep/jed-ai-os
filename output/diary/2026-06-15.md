### 💊 Medic_Station — หน้าคลังยา: เปลี่ยนรูปแบบรหัสยา + บังคับเลือกหมวดตอนเพิ่มยา (ตามที่ Jed ขอ)
- **[Jed]** ดูหน้าคลังยาแล้วถาม → Forge เสนอ 2 จุด (1. เปลี่ยนรหัสยาให้สัมพันธ์กับหมวดระบบโรค 2. เพิ่ม logic แจ้งเตือนสต็อกต่ำ) → Jed สั่งแก้ทั้ง 2 จุด + เพิ่มเงื่อนไข: ตอนเพิ่มยาใหม่ให้ผู้เพิ่มระบุหมวด (R/C/G/U/O/I/D/M) แล้ว run เลขรหัสต่อจากเดิมในหมวดนั้นอัตโนมัติ
- **[Forge]** แก้:
  1. `constants.py`: `MEDICINE_CATEGORIES` รวมทุกหมวด (เพิ่ม "M" - อื่นๆ/ทั่วไป กลับเข้ามาด้วย เพราะยาทั่วไปอย่างพาราเซตามอลเข้าหมวดนี้)
  2. `database.py`: เพิ่ม `next_medicine_code(category_code)` (รูปแบบ `<หมวด>-NNN` เช่น R-003) + migration ครั้งเดียวเปลี่ยนรหัสยาเดิม M001-M008 → รหัสใหม่ตามหมวดที่ Forge ประเมิน (M-001 พาราเซตามอล, O-001 ไอบูโพรเฟน, R-001 CPM, G-001 ORS, R-002 ยาแก้ไอ, I-001 แอลกอฮอล์, I-002 พลาสเตอร์, G-002 ยาธาตุน้ำขาว) — **Jed ควรเช็คหมวดที่ assign ให้อีกรอบ เป็นการกะคร่าวๆของ Forge**
  3. `modules/medicine.py` (MedicineDialog): ช่อง "หมวดระบบโรค" เป็น required (ค่าเริ่มต้น "-- เลือกหมวด --" กดบันทึกไม่ได้ถ้ายังไม่เลือก), ช่อง "รหัสยา" เปลี่ยนเป็น read-only auto-generate ตามหมวดที่เลือก (`db.next_medicine_code`) — ตอนแก้ไขยาเดิม ถ้าไม่เปลี่ยนหมวด รหัสเดิมจะคงไว้ ถ้าเปลี่ยนหมวดจะ gen รหัสใหม่ในหมวดนั้น
  4. `seed_data.py`: ปรับให้สร้างรหัสยาผ่าน `next_medicine_code` ตามหมวดเช่นกัน (สำหรับ DB ใหม่)
  5. logic แจ้งเตือนสต็อกต่ำ (คงเหลือ ≤ ขั้นต่ำ → "⚠️ ใกล้หมด", หมด → "⚠️ หมด") **มีอยู่แล้วในโค้ดปัจจุบัน** ไม่ต้องแก้เพิ่ม
- ตรวจสอบ: `py_compile` ผ่านทุกไฟล์, รัน migration จริงบน DB ของ Jed สำเร็จ (8 รายการเปลี่ยนรหัส+หมวดแล้ว), เปิด GUI จริงทดสอบหน้าคลังยา + ฟอร์มเพิ่มยา (เลือกหมวด R → รหัสเป็น R-003 อัตโนมัติ, ไม่เลือกหมวด → รหัสว่าง + กดบันทึกไม่ได้) ผ่านหมด ปิดโดยไม่บันทึกข้อมูลทดสอบ
