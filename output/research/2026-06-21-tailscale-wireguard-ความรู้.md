# ความรู้: Tailscale + WireGuard

วันที่: 2026-06-21
ผู้สนทนา: Jed กับ Laura

---

## คำถาม: อธิบายการใช้งาน tailscale

**Tailscale คืออะไร**
เป็น VPN แบบ mesh ที่สร้างเครือข่ายส่วนตัว (private network) เชื่อมอุปกรณ์ของเราเข้าด้วยกัน โดยไม่ต้องเปิด port หรือตั้งค่า router ให้ยุ่งยาก — ใช้เทคโนโลยี WireGuard ข้างใน

**หลักการทำงานแบบง่าย**
- ติดตั้ง Tailscale บนแต่ละเครื่อง (PC, มือถือ, server) แล้ว login ด้วยบัญชีเดียวกัน (เช่น Google account)
- แต่ละเครื่องจะได้ IP ส่วนตัว (เช่น 100.x.x.x) ที่เข้าถึงกันได้เหมือนอยู่ในเครือข่ายเดียวกัน ไม่ว่าจะอยู่คนละที่กันแค่ไหน
- ไม่ต้องเปิด port forward หรือยุ่งกับ firewall — Tailscale จัดการ NAT traversal ให้เอง

**กรณีของ Jed**
ใช้ Tailscale ดู Dashboard ผ่าน local server แทนการเปิด GitHub Pages public ทำให้:
- เข้าถึง dashboard/เว็บที่รันบนเครื่อง local ได้จากมือถือ/เครื่องอื่น โดยไม่ต้อง deploy ขึ้น public internet
- ปลอดภัยกว่า เพราะข้อมูลไม่หลุดออกไปนอกเครือข่ายส่วนตัว

**วิธีใช้งานพื้นฐาน**
1. ติดตั้ง Tailscale บนเครื่องที่รัน server (PC) และบนมือถือ/เครื่องที่ต้องการเข้าดู
2. Login ด้วยบัญชีเดียวกันทั้งสองเครื่อง
3. รัน local server บนเครื่อง PC
4. จากมือถือ เปิด browser แล้วเข้า `http://<tailscale-ip-ของ-PC>:<port>` ก็จะเห็น dashboard เหมือนอยู่ในเครือข่ายเดียวกัน

## คำถาม: เทคโนโลยี WireGuard คืออะไร ส่วนใหญ่ใช้ประโยชน์อย่างไร

**WireGuard คืออะไร**
เป็นโปรโตคอล VPN ตัวหนึ่ง (คล้าย OpenVPN หรือ IPsec) แต่ออกแบบใหม่ให้เรียบง่ายและเร็วกว่ามาก โค้ดมีแค่ประมาณ 4,000 บรรทัด (เทียบกับ OpenVPN ที่หลายแสนบรรทัด) ทำให้ตรวจสอบความปลอดภัยได้ง่ายกว่า และบั๊กน้อยกว่า

**หลักการทำงานแบบง่าย**
- สร้าง "ท่อเข้ารหัส" (encrypted tunnel) ระหว่าง 2 เครื่อง โดยใช้กุญแจเข้ารหัสสาธารณะ-ส่วนตัว (public/private key) คล้ายกับ SSH key
- ข้อมูลที่วิ่งผ่านท่อนี้จะถูกเข้ารหัสหมด ใครดักจับก็อ่านไม่ออก
- เบากว่า VPN แบบเก่า เพราะทำงานอยู่ใน kernel โดยตรง ทำให้เร็วและกินแบตน้อยกว่า

**ใช้ประโยชน์อย่างไรบ้าง**
1. **VPN ส่วนตัว** — เชื่อมต่อกลับเข้าเครือข่ายบ้าน/ออฟฟิศจากนอกบ้าน อย่างปลอดภัย
2. **Mesh network ระหว่างอุปกรณ์** — แบบที่ Tailscale ใช้ คือเอา WireGuard มาเป็นแกนกลาง แล้วเพิ่มระบบจัดการ key/login ให้ใช้งานง่ายขึ้น
3. **เข้าถึง server/NAS ที่บ้านจากนอกบ้าน** — เช่นดู dashboard ที่รันบน PC จากมือถือ
4. **ป้องกันการดักข้อมูลบน WiFi สาธารณะ** — เข้ารหัสทุกอย่างที่ส่งผ่านเน็ตคาเฟ่/ร้านกาแฟ
5. **เชื่อมต่อ server หลายเครื่อง/หลาย cloud ให้คุยกันแบบปลอดภัย** — ใช้ในงาน DevOps/infra

**สรุป:** WireGuard = เทคโนโลยีเข้ารหัสที่อยู่ข้างใน, Tailscale = บริการที่เอา WireGuard มาทำให้ใช้งานง่าย (จัดการ key, NAT, login ให้อัตโนมัติ)

## คำถาม: แต่ละเครื่องต้องต่อ internet ไหม

**ต้องต่อ internet** ทั้งสองเครื่อง (อย่างน้อยตอนเริ่มเชื่อมต่อ)

1. **ตอน login/ตั้งค่าครั้งแรก** — ต้องต่อ internet เพื่อติดต่อ Tailscale coordination server เพื่อยืนยันตัวตนและแลก public key กับเครื่องอื่น
2. **ตอนใช้งานจริง** — ส่วนใหญ่ยังต้องมี internet เพราะ:
   - ถ้าสองเครื่องอยู่คนละเครือข่าย ข้อมูลต้องวิ่งผ่าน internet เพื่อหาทางเชื่อมกัน (NAT traversal)
   - Tailscale ใช้ DERP relay server ช่วยส่งข้อมูลกรณีเชื่อมต่อตรงไม่ได้
3. **ข้อยกเว้น** — ถ้าทั้งสองเครื่องอยู่ใน LAN เดียวกัน Tailscale จะพยายามเชื่อมตรงผ่าน local network ได้ แต่ยังต้องมี internet อย่างน้อยตอนแรกเพื่อ login และเช็คสถานะเป็นระยะ

**สรุป:** Tailscale ไม่ใช่เครือข่ายแบบออฟไลน์ล้วนๆ — พึ่ง internet เป็นตัวกลางช่วย "จับคู่" และยืนยันตัวตนเสมอ ถึงแม้บางครั้งข้อมูลจริงจะวิ่งตรงในเครือข่ายเดียวกันก็ตาม

## คำถาม: ถ้า 2 เครื่องอยู่วง LAN เดียวกัน ต้องต่อเนตไหม

**ส่วนที่ยังต้องพึ่ง internet เสมอ**
- ตอน login ครั้งแรกของแต่ละเครื่อง — ต้องยืนยันตัวตนผ่าน Tailscale server
- เช็คสถานะเป็นระยะ (coordination) — เครื่องคุยกับ control server เรื่อยๆ เพื่ออัปเดต key, รายชื่อเครื่อง, ACL

**ส่วนที่ทำงานได้แม้เน็ตหลุดชั่วคราว**
- ถ้าสองเครื่อง **เคย login และเชื่อมต่อกันมาแล้ว** (key แลกกันแล้ว) แล้วอยู่ใน LAN เดียวกัน — Tailscale จะใช้ local discovery หาเครื่องอีกฝั่งได้ และส่งข้อมูลตรงผ่าน LAN โดยไม่ต้องออก internet
- กรณีนี้ถึงเน็ตที่บ้านหลุดชั่วคราว (router ต่อ ISP มีปัญหา) แต่ WiFi ภายในบ้านยังทำงาน สองเครื่องก็ยังคุยกันได้

**สรุป**
- ครั้งแรกที่ตั้งค่า → ต้องมีเน็ต 100%
- หลังจากนั้น ถ้าอยู่ LAN เดียวกันและเคยเชื่อมกันแล้ว → ใช้งานต่อได้แม้เน็ตหลุดชั่วคราว แต่ไม่การันตี 100% เพราะ Tailscale ไม่ได้ออกแบบให้เป็น offline-first

## คำถาม: ถ้าอยู่ในระบบ LAN เดียวกันแต่คนละวง (subnet/VLAN)

**ทำไมต่างจากเคส "วงเดียวกัน"**
- Local discovery ของ Tailscale (mDNS/broadcast) ทำงานได้ดีเฉพาะตอนสองเครื่องอยู่ subnet/broadcast domain เดียวกัน
- ถ้าคนละวง (เช่น VLAN10 กับ VLAN20, หรือ 192.168.1.x กับ 192.168.2.x) สองเครื่องจะมองไม่เห็นกันแบบ local โดยตรง ต้องผ่าน router/L3 switch ที่ route ระหว่างวงให้

**ผลกับ Tailscale**
- ถ้า router/switch route ข้ามวงได้และไม่บล็อก → Tailscale ยังเชื่อมตรงกันได้ผ่าน routing ภายใน ไม่ต้องออก internet จริง
- ถ้า router/firewall บล็อกการข้ามวง (ปกติในองค์กรที่แยก VLAN เพื่อความปลอดภัย) → Tailscale เชื่อมตรงไม่ได้ จะ fallback ไปใช้ DERP relay ผ่าน internet แทน

**สรุป**
- คนละวงแต่ route ถึงกันได้ภายใน → ไม่ต้องพึ่ง internet
- คนละวงและ route ไม่ถึง/ถูกบล็อก → ต้องพึ่ง internet ผ่าน DERP relay

หลักสำคัญ: Tailscale ไม่สนว่า "อยู่ระบบ LAN เดียวกัน" ขนาดองค์กรไหม สนใจแค่ว่าแพ็กเก็ตเดินทางถึงกันได้ไหม

## คำถาม: เช็คให้ว่าตอนนี้ใช้ DERP relay หรือเชื่อมตรงสำหรับ dashboard อยู่ไหม

เช็คด้วยคำสั่ง `tailscale status` บนเครื่อง PC — ผลคือ **Tailscale บนเครื่องนี้สถานะ "stopped"** (ปิดอยู่)

วิธีเช็คในอนาคต: เปิด Tailscale แล้วรัน `tailscale status` อีกครั้ง จะเห็นบรรทัดแสดง `relay "xxx"` ถ้าใช้ relay หรือ direct connection ถ้าเชื่อมตรง

## คำถาม: ผมสามารถแชร์อะไรจากเครื่องนี้ผ่าน Tailscale ไปยัง iPad ได้บ้าง

1. **Local web server / Dashboard** — รัน server บน PC แล้วเปิดผ่าน browser บน iPad ด้วย Tailscale IP (`http://100.x.x.x:port`)
2. **Taildrop** — ส่งไฟล์ตรงจาก PC ไปยัง iPad ผ่านแอป Tailscale (คล้าย AirDrop ข้าม platform)
3. **SSH / Remote access** — iPad ใช้แอป SSH client (เช่น Termius) SSH เข้า PC ผ่าน Tailscale IP
4. **Remote Desktop** — RDP/VNC ผ่าน Tailscale IP ดูหน้าจอ PC เต็มรูปแบบ
5. **Network drive / Samba** — เข้าถึง folder ผ่าน Tailscale IP จากแอป Files บน iPad (รองรับ SMB)
6. **API/Service ภายในเครื่อง** — service อื่นที่รันบน port เฉพาะ (database admin UI, n8n ฯลฯ)

**สิ่งที่ต้องมีก่อนใช้งานได้**
1. ติดตั้ง Tailscale บน PC และบน iPad (App Store)
2. Login ด้วยบัญชีเดียวกันทั้งสองเครื่อง
3. เลือกว่าจะแชร์อะไร (ไฟล์เดี่ยว → Taildrop, ไฟล์เยอะ/โฟลเดอร์ → SMB, หน้าเว็บ → local server)
